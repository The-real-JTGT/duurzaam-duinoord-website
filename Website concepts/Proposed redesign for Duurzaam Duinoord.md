# Proposed Redesign for Duurzaam Duinoord

This document turns the current website and the uploaded theme documents into one clear, theme-first website concept.

## Summary

The proposed redesign reorganises the site into four clear theme hubs:

- Energie
- Circulaire economie
- Mobiliteit
- Duurzame leefomgeving

The site should still keep a few permanent cross-site routes:

- Agenda
- Doe mee
- Over ons

The main shift is simple:

- from a menu based on what the organisation offers today
- to a menu based on the sustainability topics residents care about

That makes the site easier to understand for new visitors, easier to maintain, and better prepared for future growth.

## Status Labels

Use these labels throughout the document:

- **Current** = already on the live site
- **Planned** = explicitly proposed in the uploaded theme documents
- **Optional** = useful future ideas, but not required at launch

## A. Overall Website Concept

### 1. Purpose of the redesigned website

The redesigned site should do four jobs at once:

- explain what sustainable living in Duinoord means in practical terms
- help residents take action
- showcase neighbourhood projects, events, and results
- recruit participation, volunteers, and community involvement

The current site already does all four, but the content is scattered across mixed sections and one-off pages. The new structure should gather everything under four recognisable themes, then support those themes with shared site-wide layers for events, participation, and organisational information.

### 2. Structural Logic

Recommended primary navigation:

- Energie
- Circulaire economie
- Mobiliteit
- Duurzame leefomgeving
- Agenda
- Doe mee
- Over ons

Recommended utility navigation:

- Nieuwsbrief
- Zoek
- English

This is more intuitive for a first-time visitor who arrives with a topic in mind, such as insulation, car sharing, clean air, or greening.

### 3. How the four themes should function

Each theme should act like a mini-site with the same internal pattern:

- theme introduction
- what this means in Duinoord
- current projects and services
- practical help and tools
- events and stories
- how to participate
- future ideas and opportunities

This consistency matters because the uploaded documents are not balanced. Energy already has the deepest live content, while Circulaire economie and especially Mobiliteit need room to grow. A consistent theme template lets the site launch with uneven content depth without feeling incomplete.

## H. Future-Proofing

### 1. Use a content model, not just pages

The site should support these content types:

- Theme page
- Service/help page
- Project page
- Event page
- Story/case page
- Knowledge article / webinar / hand-out
- Form page
- Team/organisation page

This lets future initiatives fit into an existing pattern instead of forcing ad hoc page creation.

### 2. Use taxonomy across the whole site

Every item should have tags for:

- theme
- status: current / planned / optional
- content type
- audience: resident / volunteer / organiser / VvE
- topic tags: insulation, air quality, autodelen, repair, biodiversity, etc.

This allows the same event or story to appear in multiple useful places without duplication.

### 3. Separate “live now” from “coming next”

A major risk with the future documents is that the site becomes full of ideas that do not yet exist. Avoid that by using three clear presentation states:

- Nu beschikbaar
- In ontwikkeling
- Ideeën voor later

That keeps ambition visible without confusing visitors.

### 4. Fix current content-quality issues during migration

The live site still contains placeholder labels such as “Titel dia”, “Schrijf uw onderschrift hier”, and “Knop”, and some spelling inconsistencies in navigation items and page labels. These should not be migrated as-is.

The new structure is the right moment to clean content, standardise terminology, and decide which pages are evergreen, which are event-only, and which are archive items.

### 5. Do not keep a half-translated English section

The new site should either:

- offer a small, intentional English layer with key pages only
- or offer full translation for selected templates

It should not continue with a partial mirror that feels unfinished.

## Bottom Line

The best redesign is not to force all content rigidly into only four buckets. It is to make the four themes the primary organising principle, while supporting them with Agenda, Doe mee, and Over ons as permanent cross-site layers.

That preserves the best of the current website, gives every existing page a logical home, and creates enough room for the future ideas in the uploaded documents to grow without making the site confusing.
